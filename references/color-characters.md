# 房语小黑色彩角色系统

## 定位

在小黑 2.0 里，彩色衣服不是装饰，也不是重新发明一批吉祥物。它是“穿着不同颜色衣服的小猫”角色家族的核心识别方式，用于表达输入内容里的语言场景、情绪状态、行动功能和叙事分工。

默认仍然以“小黑”为主角。只有当内容里有明确的情绪转折、系统角色、风险提醒、修复动作或机会节点时，才引入彩色小黑。

最重要的视觉规则：身体永远是黑色，衣服颜色负责区分角色。不要把它理解成彩色身体的小人。

视觉标准锚点：`assets/examples/color-character-family-cat-clothing.png`。当角色形体、服装比例或颜色关系不确定时，优先回看这张图。

核心原则：

```text
同一猫耳形体，同一黑色身体，同一眼睛，同一动作逻辑，只改变工作服颜色。
```

必须保持：

- 哑光黑色身体、近圆脑袋、两只小三角猫耳、小紧凑身体，穿简洁的低饱和彩色工作服。
- 两个小白点或小白椭圆眼睛。
- 无嘴或极简短线嘴。
- 细胳膊、细腿。
- 冷静、认真、空空的表情。
- 必须和真实物件发生物理动作。
- 工作服简洁、无 Logo、无文字；不戴帽子、不加动物装饰。

禁止：

- 把彩色小黑做成单独吉祥物。
- 用发型、表情、饰品或复杂服装区分角色；只允许用简洁工作服的颜色表达语义。
- 因为衣服颜色变化而改变比例、猫耳、眼睛、四肢和性格。
- 一张图塞入太多彩色角色，导致画面像角色海报。

## 五色角色

### 小黑：默认 / 主叙事 / 观察者

衣服颜色：哑光黑色工作服。

适合：默认主角、职场荒诞、AI 工具、流程长卷、项目复盘、复杂长卷主线。

提示词短句：

```text
matte black Xiaohei, default main operator
```

### 小红：警报 / 冲突 / 情绪高点

颜色：低饱和深番茄红 / 暗红，不能鲜艳儿童红。

适合：风险、报警、热度、冲突、争夺、突然放量、大涨、舆论爆点、产品商战、限额放开、竞争动作。

典型动作：拉警报绳、举警示牌、敲红章、拉住红线、按下红色按钮。

提示词短句：

```text
muted deep tomato-red Xiaohei for alarm, tension, conflict, emotional peak
```

### 小绿：修复 / 回落 / 恢复 / 通过

颜色：低饱和深草绿 / 墨绿。

适合：修复、恢复、通过、回归理性、风险释放、股市语境里的下跌/回落/割肉、资源连接、关系恢复、流程打通。

典型动作：缠胶带、接线、补洞、托住下坠物、把断开的线接回去。

提示词短句：

```text
muted deep grass-green Xiaohei for repair, recovery, pass-through, market drop in Chinese stock context
```

### 小蓝：系统 / 理性 / 工具 / 数据

颜色：低饱和钴蓝 / 深蓝灰。

适合：技术、代码、数据、系统、工具、自动化、配置、AI 工作流、开发者场景、产品逻辑、理性分析。

典型动作：拧旋钮、接插件、调仪表、拉线缆、整理终端线。

提示词短句：

```text
muted cobalt-blue Xiaohei for system, tools, data, rational operation
```

### 小黄：机会 / 提醒 / 预热 / 灵感

颜色：低饱和芥末黄 / 暖黄，不要荧光黄。

适合：机会窗口、灵感、活动预热、提醒、报名、启动、点亮、下一步。

典型动作：托灯泡、贴提醒点、点亮小灯、拉开帷幕、递出钥匙。

提示词短句：

```text
muted mustard-yellow Xiaohei for opportunity, reminder, warm-up, idea spark
```

## 颜色路由

### 单色优先

标准 16:9 正文配图默认只用一个小黑角色。

使用单色彩色小黑的条件：

- 内容的核心情绪或功能非常明确。
- 颜色能帮助 3 秒读懂，而不是增加解释负担。
- 画面只有一个核心物理动作。

### 组合使用

只有在内容里存在多个明确角色分工时，才组合使用彩色小黑。

组合上限：

- 标准 16:9：最多 2 个小黑角色，极少数可到 3 个。
- 彩蛋长卷：每个节点最多 1 个主角色；全图可出现 3-5 种颜色，但必须跟节点语义绑定。

常见组合：

- 小黑 + 小红：普通用户被警报/冲突拉扯。
- 小蓝 + 小红：系统操作遇到风险报警。
- 小蓝 + 小绿：工具链修复、流程恢复。
- 小黑 + 小黄：活动启动、机会提醒、预热。
- 小红 + 小绿：涨跌、冲突后的修复、情绪高低对照。
- 小黑 + 小蓝 + 小红：用户、系统、风险三方关系。

### 长卷节点配色

彩蛋长卷的颜色要服务阶段记忆，不要平均撒色。

建议：

- 起点 / 主叙事：小黑。
- 启动 / 预热 / 点亮：小黄。
- 技术 / 工具 / 系统化：小蓝。
- 冲突 / 高峰 / 商战 / 报警：小红。
- 修复 / 收束 / 资源连接 / 恢复：小绿。

如果节点很多，不要每个节点都换色。优先让主线小黑贯穿，关键节点出现彩色角色。

## Shot List 必填

做 shot list 时增加：

```text
角色颜色：
选择理由：
组合关系：
颜色负面约束：
```

示例：

```text
角色颜色：小蓝 + 小红
选择理由：小蓝代表系统调参，小红代表限额放开背后的竞争压力。
组合关系：小蓝拧额度旋钮，小红拉警报绳，黑色线缆把两者连到同一个用量仪表。
颜色负面约束：不改变小黑形体，衣服保持简洁无标志，不把红蓝做成阵营海报。
```

## Prompt 规则

在提示词里明确写：

```text
Color character routing:
Use {角色颜色} because {选择理由}. All Xiaohei variants share the same exact black body, cat ears, tiny white eyes, thin limbs, blank serious expression, and the same physical-action logic. Only the matte work-shirt color changes.
```

如果使用组合角色，还要写清每个角色的动作，不要只说“出现几个颜色的小黑”。

## QA

检查：

- 颜色是否来自输入内容的语义，而不是装饰。
- 颜色是否帮助 3 秒读懂。
- 所有彩色角色是否保持同一形体、同一眼睛和同一动作逻辑。
- 标准模式是否没有超过 2 个角色；如果超过，是否真的必要。
- 彩蛋长卷是否没有把每个节点都机械换色。
- 彩色角色是否仍然在和真实物件发生物理动作。
- 画面是否仍然是小黑 2.0，而不是角色设定海报。
