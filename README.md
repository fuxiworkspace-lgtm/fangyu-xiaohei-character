# 房语小黑：让内容穿上颜色，开始行动

🌐 **Language 

语言：** [中文](README.md) · [English](README.en.md)

`fangyu-xiaohei-character` 是一套面向中文/英文内容创作的角色视觉 Skill。

它把文章、观点、项目复盘、活动流程和个人经历，变成一组有角色、有动作、有情绪记忆点的视觉场景：

> 黑色猫耳小猫，穿上不同颜色的工作服，进入真实物件现场，认真完成一件荒诞但准确的工作。

这里的颜色不是装饰，衣服也不是道具。工作服颜色决定小猫在故事里的身份：谁在报警，谁在修复，谁在调系统，谁在点亮机会。真实物件和物理动作，则让抽象内容变成读者一眼就能读懂的现场。

## 为什么值得用

- 把抽象观点变成具体动作，不再停留在概念解释。
- 用工作服颜色建立稳定的角色语义，适合持续创作和系列化表达。
- 兼容 16:9 正文图、双角色协作、3:1 长卷和 4:5 社交卡。
- 自带角色锁定、颜色路由、Prompt 模板、视觉锚点和质量检查。
- 适合公众号、社交媒体、项目复盘、活动流程、产品故事和 AI 内容创作。

## 核心角色设定

所有角色都共享同一套黑色猫耳身体：

- 黑色哑光身体
- 近圆形大脑袋
- 两只小而克制的三角猫耳
- 小而紧凑的身体
- 白色椭圆眼睛和小黑色瞳孔
- 短小四肢
- 空、呆、冷静、认真的表情
- 不加尾巴、胡须、爪垫等写实动物细节

角色差异来自工作服，而不是身体颜色。工作服必须简洁、哑光、低饱和、无 Logo、无文字、无复杂图案。

## 工作服颜色系统

| 角色 | 工作服颜色 | 角色语义 | 常见动作 |
| --- | --- | --- | --- |
| 小黑 | 墨黑 | 主叙事、观察、默认操作员 | 推纸堆、整理资料、承载主线 |
| 小红 | 陶土红 | 警报、冲突、风险、情绪高点 | 拉警报绳、挡住红线、敲响警铃 |
| 小绿 | 苔藓绿 | 修复、恢复、连接、通过 | 接线、缠胶带、补洞、托住下坠物 |
| 小蓝 | 岩石蓝 | 系统、工具、数据、理性操作 | 拧旋钮、调仪表、接插件、整理线缆 |
| 小黄 | 芥末黄 | 机会、提醒、预热、启动、点亮 | 开灯、递钥匙、拉开帷幕、打开入口 |

最重要的判断顺序：

1. 先判断小猫在内容里承担什么角色。
2. 再决定它应该穿什么颜色的工作服。
3. 最后设计真实物件和物理动作。

不要把它生成成“身体是彩色的小人”。身体始终是黑色，工作服颜色是第一视觉识别信号。

## 画面版式

### 16:9 单角色

适合表达一个明确的情绪或观点。默认一个小猫、一个真实主物件、一个核心物理动作和少量手写标签。

### 16:9 双角色

只在内容存在清晰分工时使用，例如：蓝色工作服负责系统，绿色工作服负责修复。标准画面最多两个角色，极少数情况下使用三个。

### 3:1 长卷

适合项目复盘、活动流程、产品演化和个人成长。使用曲线路径、5-8 个真实物件节点和阶段性服装颜色，形成自然的视觉叙事。

### 4:5 竖版

适合社交媒体单观点卡片。聚焦一个工作服颜色、一个真实物件和一个清晰动作。

## 使用方式

将本目录安装为 Codex Skill 后，使用 `$fangyu-xiaohei-character` 调用。

```text
使用 $fangyu-xiaohei-character，把这段关于 AI 竞争的内容做成一张 16:9 配图。让小红穿陶土红工作服拉警报绳，小蓝穿岩石蓝工作服调系统旋钮。
```

```text
使用 $fangyu-xiaohei-character，把“活动预热到大会闭幕”做成一张 3:1 长卷。启动节点用小黄工作服，正式大会用小黑工作服，资源协同和收束节点用小绿工作服。
```

```text
使用 $fangyu-xiaohei-character，把这段内容做成 4:5 竖版社交卡，只使用小黄工作服小猫，通过点灯和递钥匙表达机会来了。
```

## 锚点使用原则

- 保持黑色猫耳身体统一。
- 用工作服颜色区分角色，不给身体换色。
- 工作服保持简洁、低饱和、无 Logo、无文字。
- 小猫必须和真实物件发生物理动作。
- 颜色必须来自内容语义，而不是装饰。
- 锚点只锁定角色和质量，不复制具体物件、布局或文字。
- 保持近白背景、真实物件、手绘质感和充足留白。
